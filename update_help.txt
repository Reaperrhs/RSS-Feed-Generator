Usage: index functions update [options]

Update function by its unique ID.

Options:
  --function-id <function-id>                          Function ID.
  --name <name>                                        Function name. Max length: 128 chars.
  --runtime <runtime>                                  Execution runtime.
  --execute [execute...]                               An array of role strings with execution
                                                       permissions. By default no user is granted
                                                       with any execute permissions. [learn more
                                                       about
                                                       roles](https://appwrite.io/docs/permissions#permission-roles).
                                                       Maximum of 100 roles are allowed, each 64
                                                       characters long.
  --events [events...]                                 Events list. Maximum of 100 events are
                                                       allowed.
  --schedule <schedule>                                Schedule CRON syntax.
  --timeout <timeout>                                  Maximum execution time in seconds.
  --enabled [value]                                    Is function enabled? When set to 'disabled',
                                                       users cannot access the function but Server
                                                       SDKs with and API key can still access the
                                                       function. No data is lost when this is
                                                       toggled.
  --logging [value]                                    When disabled, executions will exclude logs
                                                       and errors, and will be slightly faster.
  --entrypoint <entrypoint>                            Entrypoint File. This path is relative to
                                                       the "providerRootDirectory".
  --commands <commands>                                Build Commands.
  --scopes [scopes...]                                 List of scopes allowed for API Key
                                                       auto-generated for every execution. Maximum
                                                       of 100 scopes are allowed.
  --installation-id <installation-id>                  Appwrite Installation ID for VCS (Version
                                                       Controle System) deployment.
  --provider-repository-id <provider-repository-id>    Repository ID of the repo linked to the
                                                       function
  --provider-branch <provider-branch>                  Production branch for the repo linked to the
                                                       function
  --provider-silent-mode [value]                       Is the VCS (Version Control System)
                                                       connection in silent mode for the repo
                                                       linked to the function? In silent mode,
                                                       comments will not be made on commits and
                                                       pull requests.
  --provider-root-directory <provider-root-directory>  Path to function code in the linked repo.
  --specification <specification>                      Runtime specification for the function and
                                                       builds.
  -h, --help                                           display help for command
